# Plan 02-03: Pipe ML results back into the WebSocket response

<objective>
Integrate the `FaceAnalyzer` into the FastAPI WebSocket loop so incoming frames are processed and result JSON is streamed back to the client.
</objective>

<action_plan>
<task>
<read_first>
- backend/main.py
- backend/ml_engine.py
</read_first>
<action>
In `backend/main.py`, import `FaceAnalyzer` from `ml_engine.py`.
Instantiate a global `analyzer = FaceAnalyzer()`.
In the `@app.websocket("/ws")` endpoint's `while True` loop, update the message handling:
Instead of just responding with `{"status": "received"}`, pass the base64 frame string to `analyzer.process_frame(data)`.
Since `process_frame` is synchronous and CPU bound, use `await asyncio.to_thread(analyzer.process_frame, data)` to prevent blocking the async loop.
Send the resulting JSON containing `emoji` and `why_meter` back to the client via `await websocket.send_json(result)`.
Handle frames where no face is detected gracefully (e.g., return `{"emoji": "none", "why_meter": {"why_score": 0}}`).
</action>
<acceptance_criteria>
- `backend/main.py` contains `asyncio.to_thread`.
- `backend/main.py` imports `FaceAnalyzer`.
- WebSocket sends JSON containing `emoji` and `why_meter`.
</acceptance_criteria>
</task>
</action_plan>
