# Plan 04-01: Build the Entry Gate

<objective>
Implement an initial splash screen that requires user interaction to enter the application, triggering fullscreen mode.
</objective>

<action_plan>
<task>
<read_first>
- frontend/src/App.tsx
</read_first>
<action>
In `frontend/src/App.css`:
Add styles for `.splash-screen` (full width/height wrapper, centered flexbox) and `.enter-btn` (large, aggressive styling).

In `frontend/src/App.tsx`:
Add a `hasEntered` state variable (boolean, default false).
Create an `handleEnter` function that:
1. Calls `document.documentElement.requestFullscreen().catch(e => console.log(e))`
2. Sets `hasEntered` to true.
If `!hasEntered`, return the splash screen `div` with the "Enter the Simulation" button.
If `hasEntered`, return the `WebcamCapture` component.
</action>
<acceptance_criteria>
- App renders Splash Screen on mount.
- Clicking Enter requests fullscreen and renders WebcamCapture.
</acceptance_criteria>
</task>
</action_plan>

<verification>
- Verify `hasEntered` state gating logic exists in `App.tsx`.
- Manual verification of fullscreen request.
</verification>

<must_haves>
- The user must click a button to start the experience.
- The browser triggers fullscreen upon entry.
</must_haves>
