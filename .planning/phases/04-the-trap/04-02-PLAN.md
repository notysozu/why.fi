# Plan 04-02: Implement Fullscreen traps and warnings

<objective>
Detect early escapes from fullscreen and present an endgame lock screen on successful completion.
</objective>

<action_plan>
<task>
<read_first>
- frontend/src/components/WebcamCapture.tsx
</read_first>
<action>
In `frontend/src/App.css`:
Add `.trap-warning` class (fixed full screen red background, high z-index, giant text "RETURN TO FULLSCREEN").

In `frontend/src/components/WebcamCapture.tsx`:
Add state `isTrapped` (boolean, default false).
Add a `useEffect` to listen to `"fullscreenchange"` on `document`. If `!document.fullscreenElement` and `gameScore < 5`, set `isTrapped(true)`.
If `isTrapped` is true, render the `.trap-warning` overlay on top of everything else, with a button to `requestFullscreen()` again. Once clicked, set `isTrapped(false)`.
For "The Escape Challenge" (UX-03):
When `gameScore >= 5`, replace the target game message with "YOU ARE FREE. YOU MAY QUIT." and render a functional "QUIT" button. Clicking it should unmount the webcam entirely (perhaps via a callback or local state `isFree` setting video stream to null).
</action>

- A functional quit button is revealed at score >= 5.
</acceptance_criteria>
</task>
</action_plan>
