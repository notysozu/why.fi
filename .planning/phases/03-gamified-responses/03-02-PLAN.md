# Plan 03-02: Build Emoji Match Game Loop

<objective>
Implement the "Match the Emoji" mini-game loop using the incoming `emoji` classification from the backend.
</objective>

<action_plan>
<task>
<read_first>
- frontend/src/components/WebcamCapture.tsx
</read_first>
<action>
In `frontend/src/App.css`, add styles for the game prompt overlay (e.g. `.game-prompt`).
In `WebcamCapture.tsx`:
Define a list of `EMOJIS = ["happy", "sad", "angry", "surprise", "neutral"]`.
Add state for `targetEmoji` (randomly select one from EMOJIS).
Add state for `gameScore` (starts at 0).
Add state for `gameMessage` (instructions like "Make this face: 😲").
In the WebSocket `onmessage` handler, extract `data.emoji`.
If `data.emoji` matches `targetEmoji`:
  - Increment `gameScore`.
  - Set `gameMessage` to "Match! Next...".
  - Use `setTimeout` (1 second) to pick a new random `targetEmoji` and reset the message to "Make this face...".
Render the `gameMessage` and `gameScore` overlaid on the screen.
</action>
<acceptance_criteria>

- `WebcamCapture.tsx` contains matching logic in `onmessage`.
- Overlay text shows the current target and score.
</acceptance_criteria>
</task>
</action_plan>
