# Plan 01-03: Implement the WebSocket connection loop (Client/Server)

<objective>
Establish a WebSocket connection to stream frames from React to FastAPI.
</objective>

<action_plan>
<task>
<read_first>
- backend/main.py
</read_first>
<action>
In `backend/main.py`, add a websocket endpoint `@app.websocket("/ws")`.
In the endpoint, `await websocket.accept()`, then enter a `while True` loop to `await websocket.receive_text()` (expecting base64 image data).
Reply with a JSON acknowledgment `{"status": "received"}` for each frame.
Handle `WebSocketDisconnect` gracefully.
</action>
<acceptance_criteria>
- `backend/main.py` contains `@app.websocket("/ws")`.
- `backend/main.py` handles `WebSocketDisconnect`.
</acceptance_criteria>
</task>

<task>
<read_first>
- frontend/src/components/WebcamCapture.tsx
</read_first>
<action>
In `frontend/src/components/WebcamCapture.tsx`, establish a `WebSocket` connection to `ws://localhost:8000/ws` inside a `useEffect`.
Add a hidden `<canvas>` element to capture frames from the video.
Set up a `setInterval` (frequency: 100ms) to draw the video to the canvas: `ctx.drawImage(video, 0, 0, 320, 240)` and extract the data: `canvas.toDataURL('image/jpeg', 0.5)`.
Send this base64 string over the WebSocket if its state is `WebSocket.OPEN`.
Clean up the interval and WebSocket on component unmount.
</action>
<acceptance_criteria>
- `frontend/src/components/WebcamCapture.tsx` contains `new WebSocket('ws://localhost:8000/ws')` or similar connection URL.
- `frontend/src/components/WebcamCapture.tsx` contains `setInterval` formatting frames.
</acceptance_criteria>
</task>
</action_plan>

<verification>

- Start the backend and frontend. Backend console shows regular frame receipt.
</verification>

<must_haves>
